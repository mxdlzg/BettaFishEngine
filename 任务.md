**总体原则**
这次只做一件事：

**基于原完整舆情分析项目，新增一层 LangGraph 包装，使其能像法律助手一样部署到 ALB 平台。**

约束也固定：

- 不修改 `ALB_Workbench`
- 不以 `skill` 为主
- 知识库通过 MCP 接入
- LLM 统一走你提供的 OpenAI 兼容服务
- 最终输出必须适配 ALB 现有 LangGraph 前端能力
  - 文本流
  - tool step
  - files 附件
  - 历史恢复

下面是最细的开发任务单。

---

**任务总览**
1. 准备与基线确认
2. 原舆情系统能力抽象
3. LangGraph 包装层搭建
4. MCP 知识库接入
5. LLM 接入与提示词体系
6. 多阶段工作流节点实现
7. 报告与附件输出
8. 部署配置
9. 联调验收
10. 运维与后续增强

**第 1 阶段：准备与基线确认**

- 验收：不影响原 Flask/FastAPI 主服务运行

2. 固化当前项目目录与入口
- 任务：记录以下模块的真实入口文件
  - `QueryEngine`
  - `MediaEngine`
  - `InsightEngine`
  - `ReportEngine`
  - `ForumEngine`
- 输出：一份内部文档，列出每个模块的主入口、依赖配置、返回结构
- 验收：后面每个节点都知道调用哪个 Python 模块/类/函数

3. 固化运行方式
- 任务：确认原项目当前可运行的最小命令
  - 主应用
  - 单独某个引擎
  - 报告生成
- 输出：命令清单
- 验收：在服务器或本地可复现运行

4. 确认模型服务
- 任务：在服务器执行：
```bash

--header 'Authorization: Bearer <你的key>'
```
- 输出：真实模型列表
- 验收：拿到可用的 `model id`
- 注意：你之前贴出的 key 建议后续轮换

5. 确认 MCP 服务
- 任务：验证 MCP 服务真正可用
- 输出：
  - 能连通 `http://192.168.195.195:18012/api/mcp`

- 验收：能成功列工具或完成一次最小 MCP 调用

**第 2 阶段：原舆情系统能力抽象**

目标：把原系统改成“可被 LangGraph 节点调用”的能力层。

6. 建立统一 bridge 层
- 任务：新增 `multi_agents/tools/engine_bridge.py`
- 目标：统一封装原系统能力调用
- 输出接口：
  - `run_query_engine(query, context)`
  - `run_media_engine(query, context)`

  - `run_forum_round(query, inputs, round_index, context)`
  - `run_report_engine(query, merged_result, context)`
- 验收：这些方法都能单独运行，不依赖 Web UI

7. 抽取 QueryEngine 调用入口
- 任务：把原 `QueryEngine` 的执行流程提炼成可调用函数
- 输入：
  - `query`
  - `time_range`
  - `focus`
  - `locale`
- 输出：
  - `summary`
  - `sources`
  - `raw_result`
  - `logs`
- 验收：单测或脚本调用可返回结构化结果

8. 抽取 MediaEngine 调用入口
- 任务：封装 `MediaEngine`
- 输出：
  - `platform_distribution`
  - `media_highlights`
  - `short_video_findings`
  - `source_urls`
  - `logs`
- 验收：给定一个主题可返回媒体分析结果

9. 抽取 InsightEngine 调用入口
- 任务：封装 `InsightEngine`
- 输出：
  - `sentiment_summary`
  - `keyword_clusters`
  - `topic_clusters`
  - `risk_signals`
  - `logs`
- 验收：给定主题能返回情感和主题结构

10. 抽取 ForumEngine 调用入口
- 任务：把论坛协作机制包装成单轮可调用函数
- 输入：
  - `query`
  - `query_result`
  - `media_result`
  - `insight_result`
  - `kb_result`
  - `round_index`
- 输出：
  - `round_summary`
  - `conflicts`
  - `missing_evidence`
  - `followup_suggestions`
- 验收：支持独立运行 1 轮讨论

11. 抽取 ReportEngine 调用入口
- 任务：把 `ReportEngine` 包成统一函数
- 输出：
  - `summary_text`
  - `html_path`
  - `pdf_path`
  - `docx_path` 或 `md_path`
  - `charts`
  - `manifest`
- 验收：输入融合后的结果，能生成完整产物

12. 统一返回格式
- 任务：为所有 bridge 返回定义统一 schema
- 输出：`dict` 标准结构说明
- 验收：所有节点不再使用各自混乱结构

**第 4 阶段：MCP 知识库接入**

18. 实现 `mcp_client.py`
- 任务：写 MCP SSE 客户端
- 支持：
  - 建立连接
  - 发送 JSON-RPC 请求
  - 解析 SSE 响应
  - 错误处理

- 请求必须包含：
  - `Authorization: Bearer <token>`
  - `Accept: text/event-stream`
- 验收：可独立列工具、调用工具

19. 实现 MCP 工具封装
- 任务：在 `multi_agents/tools/` 内再写一个知识库包装，例如 `kb_mcp_tool.py`
- 对外接口：
  - `list_knowledge_bases(token)`
  - `query_knowledge_base(token, kb_id, query, mode, top_k)`
- 验收：给定 token 能查到 KB 和查询结果

20. 设计 token 读取优先级
- 任务：Agent 侧优先使用 thread metadata 里的 `alb_mcp_token`
- 回退：
  - 如果没有，再使用 `.env` 的 fallback token
- 验收：在 ALB 平台里运行时自动用用户上下文 token

21. 实现自动选库逻辑
- 任务：因为你不改 ALB 前端，Agent 必须自己选知识库
- 流程：
  - 先 `list_knowledge_bases`
  - 再用 LLM 挑选 1~3 个最相关库
  - 再执行 query
- 输出：
  - `auto_selected_knowledge_bases`
  - `kb_result`
- 验收：用户不显式选库时仍能稳定工作

22. 实现知识库结果标准化
- 任务：把 MCP 返回转成统一结构
- 至少包含：
  - `knowledge_base_id`
  - `knowledge_base_name`
  - `answer`
  - `sources`
  - `citations`
  - `confidence`
- 验收：后续 merge 节点可直接用

from langgraph.graph import StateGraph, START, END
from multi_agents.state import PublicOpinionState
from multi_agents.nodes.intake import intake_node
from multi_agents.nodes.planner import planner_node
from multi_agents.nodes.query_engine_node import query_engine_node
from multi_agents.nodes.media_engine_node import media_engine_node

from multi_agents.nodes.finalize_node import finalize_node

builder = StateGraph(PublicOpinionState)

builder.add_node("intake", intake_node)
builder.add_node("planner", planner_node)
builder.add_node("query_engine", query_engine_node)
builder.add_node("media_engine", media_engine_node)
builder.add_node("insight_engine", insight_engine_node)
builder.add_node("kb_mcp", kb_mcp_node)
builder.add_node("forum_round_1", forum_round_1_node)
builder.add_node("forum_round_2", forum_round_2_node)
builder.add_node("forum_round_3", forum_round_3_node)
builder.add_node("merge", merge_node)
builder.add_node("report", report_node)
builder.add_node("artifacts", artifacts_node)
builder.add_node("finalize", finalize_node)

builder.add_edge(START, "intake")
builder.add_edge("intake", "planner")
builder.add_edge("planner", "query_engine")
builder.add_edge("query_engine", "media_engine")
builder.add_edge("media_engine", "insight_engine")
builder.add_edge("insight_engine", "kb_mcp")
builder.add_edge("kb_mcp", "forum_round_1")
builder.add_edge("forum_round_1", "forum_round_2")
builder.add_edge("forum_round_2", "forum_round_3")
builder.add_edge("forum_round_3", "merge")
builder.add_edge("merge", "report")
builder.add_edge("report", "artifacts")
builder.add_edge("artifacts", "finalize")
builder.add_edge("finalize", END)

graph = builder.compile()
